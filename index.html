Areas de GIT

Working Directory

El directorio de trabajo hace referencia a la carpeta donde creamos el proyecto, muy rara vez utilizaremos la linea de comandos y trabajaremos principalmente con nuestro editor predilecto.

Staging Area (INDEX)

El area de control de cambios corresponde a un Archivo de la carpeta .git que identifica cuales son aquellos elementos del proyecto a los que estamos haciendole seguimiento. Es el area donde se realiza la captura del codigo (SNAPSHOT) para ser confirmada mas adelante.

Repository (LOCAL/REMOTE)

Es el area donde se almacenan los cambios confirmados mediante commits en formato BLOB (Binary Large Object). Basicamente es lo nos permite acceder a puntos anteriores dentro del proyecto, ya que podemos apuntar a los diferentes nodos dentro del historial de confirmaciones.
